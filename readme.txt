2.写作方式
首先明确小节的主题
然后列出具体涉及的知识点
具体写知识点时注意以下事项：

    
    1.
    
    遵循先讲最常用，最实用的功能。
    
    高级用法，细节区别，可以提及，但是不需要展开。
    
    在列举知识点时就需要注意这一点。

    因为这本书的定位不是对某个工具进行深入剖析，而是让读者对网站开发涉及的方方面面树立基本的概念。
    
    重点在于让读者知道有可以这样去做，有这样的工具，以及这些工具最基本也是最常见的用法。

    简单是说，本书的目的在于“师傅领进门”，至于修行那要看个人了。

    2.
    
    语言尽量通俗，避免抽象的概念和描述,不要纠结于描述的精确性 --- 这毕竟不是发Paper。
    
    有一个简单的技巧就是让句子尽可能的短：）
    
    举例
    “
    版本管理是管理在电脑上的文档，程序和其它信息的一种机制。
    
    它把每次改动作一个编号，叫做版本号。每次提交一个新的改动的时候，版本管理系统会新建一个版本号，记录下每次的改动的内容和对应的修改时间，修改人等信息。
    
    除此之外，版本管理通常会支持比较，恢复，合并等操作。
    
    专业的软件开发都需要使用版本管理工具来管理开发的源码和文档，这样的好处有很多，最主要的是可以放心大胆地修改代码，万一误入歧途，可以随时回到之前的时间点，另外，对于多人协同工作，可以方便做代码比较和合并。
    
    ”

    这样写就是很像乏味无趣的教科书，让人犯困。

    可以改为下面这样

    “
    照相，能帮我们记录下那些美好的旧时光。

    版本管理，就是好像对文件照相。

    有了版本管理，我们就可以轻而易举的“Yesterday , Once More”
    
    另外，写代码，很多时候并不是一个人在战斗。

    这时候，就需要常常同步各自的改动。

    复制来复制去，麻烦不说，还经常一不小心，就把已有的改动覆盖掉。

    而有了版本管理，无痛同步，是那么的写意自然。
    ”

    3.
    链接用官方的链接
    比如
    """
    他发现最好用的客户端是Tortoisesvn，于是他去http://dl.pconline.com.cn/download/53122.html下载了一个，一路下一步安装起来。
    """
    
    首先这一段话逻辑有一点小问题，“发现最好用的客户端” -- 没有装怎么发现
    可以把“发现”改为“听说”
    
    如下
    """
    他听说最好用的客户端Tortoise Svn ,摸索到 http://tortoisesvn.net/downloads ，下载了软件和汉化包，一路Next便安装成功。

    下载Tortoise Svn svn_down.jpg
    """
    其中第二行 svn_down.jpg是配的截图，截取前面配的名字也应该规范，因为他会显示在图片下方。
    
    4.
    配合描述，多截图
    截图可以反映的，就可以不用重复（可以再截图中配上文字）
    比如
    
    提交代码需要将要提交的文件放入代码存放目录，点击右键，选择svn update，在窗口上面的消息输入框输入这次改动的内容描述，在窗口下面的改变文件列表中选择要提交的文件，点击确定。文件就被提交了。
    
    可以精简为，

    点击右键，选择 SVN提交，勾选需要的文件，即可上传改动。如图：



    5.英文名拼写的问题，统一采用这种格式
    Google Code
    Tortoise Svn
    Mozilla Firefox
